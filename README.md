Scan Your Local Network for Open Ports

Objective
This project focuses on discovering active devices and identifying open ports in a local network using Nmap. It also includes packet analysis using Wireshark to understand network communication and security exposure.

Tools Used
Nmap
Wireshark
Kali Linux

Procedure
The system IP address was identified using the ip a command. Based on the IP address 192.168.59.129, the network range 192.168.59.0/24 was determined.

A TCP SYN scan was performed using the command sudo nmap -sS 192.168.59.0/24 to discover active hosts and open ports.

A detailed scan was also performed using sudo nmap -sS -sV -O 192.168.59.0/24 and the output was saved in a file for further analysis.

Wireshark was used to capture network packets during the scan. The filter tcp.flags.syn == 1 was applied to observe SYN packets generated by the scan.

Results
Four active hosts were identified in the network. Most of the ports were found to be filtered, indicating the presence of firewall protection. One host showed port 53 in a filtered state, which is associated with DNS service. No open ports were detected in the network.

Analysis
The absence of open ports indicates that no services are publicly exposed. Filtered ports suggest that firewall rules are actively blocking incoming connections. Some closed ports were identified through reset responses. The network environment appears to be a virtual network using VMware.

Security Risks
Open ports can allow unauthorized access if not properly secured. Services running on exposed ports may contain vulnerabilities. Misconfigured DNS services can be exploited. However, in this network, most ports are filtered which reduces the attack surface.

Wireshark Findings
Wireshark captured SYN packets sent from the source system to multiple hosts. Most packets did not receive responses, indicating filtered ports. Some reset packets were observed, indicating closed ports. No SYN ACK responses were seen, confirming that no open ports were present.

Learning Outcome
Understanding of network reconnaissance techniques
Knowledge of TCP SYN scanning
Experience using Nmap and Wireshark
Understanding firewall behavior and port filtering

Conclusion
The network scan showed that all devices are well protected with firewall configurations. No open ports were detected, and most ports were filtered, reducing the risk of unauthorized access. Wireshark analysis confirmed the behavior of TCP SYN scanning and validated the scan results.
